Данный документ содержит функциональные требования к системе управления логистическими операциями (CRM). Спецификация определяет поведение системы с точки зрения пользователя, бизнес-правила, ограничения и критерии приёмки.

1. Рабочий стол (Dashboard)
1.1 Назначение модуля
Единая точка входа в систему, предоставляющая персонализированную информацию о текущих задачах, активных операциях и уведомлениях. Рабочий стол адаптируется под роль пользователя, показывая релевантные виджеты и метрики.
1.2 Экраны
E-1.2.1 Главный рабочий стол
Описание: Первичный экран после авторизации, содержащий набор персонализированных виджетов.
Структура экрана:
┌─────────────────────────────────────────────────────────────────────┐
│  HEADER: Logo | Поиск [_______] | 🔔 (3) | 👤 User ▼               │
├─────────────────────────────────────────────────────────────────────┤
│ ┌─────────────────┐ ┌─────────────────┐ ┌─────────────────────────┐ │
│ │  МОИ ЗАДАЧИ     │ │ АКТИВНЫЕ РЕЙСЫ  │ │  УВЕДОМЛЕНИЯ            │ │
│ │  ─────────────  │ │  ─────────────  │ │  ─────────────────────  │ │
│ │  • Заявка #123  │ │  🚛 Рейс #456   │ │  🔔 Новая заявка #789   │ │
│ │  • Заявка #124  │ │    Москва→СПб   │ │  🔔 Просрочен срок      │ │
│ │  • Рейс #125    │ │    В пути 2ч    │ │  🔔 Назначен рейс       │ │
│ │                 │ │                 │ │                         │ │
│ └─────────────────┘ └─────────────────┘ └─────────────────────────┘ │
│ ┌───────────────────────────────────────────────────────────────────┐│
│ │  БЫСТРЫЕ ДЕЙСТВИЯ                                              ││
│ │  [+ Заявка] [+ Рейс] [📋 Отчёты] [👥 Клиенты]                 ││
│ └───────────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────────┘
Видимые виджеты по ролям:

Мои задачи	✅

KPIs дня	✅

Последние уведомления	✅
Статистика загрузки флота	✅
E-1.2.2 Детальный виджет
Описание: Раскрывающаяся секция виджета с полным списком элементов.
Возможности:
- Пагинация (20 элементов на страницу)
- Сортировка по дате, приоритету
- Быстрые фильтры
- Массовые действия (выделить несколько)

A-1.3.2	Открытие объекта
A-1.3.3	Изменение виджетов
A-1.3.4	Поиск глобальный
A-1.3.5	Просмотр уведомлений
A-1.3.6	Быстрые действия
1.4 Бизнес-правила
ID	Правило
BR-1.4.1	Персонализация
BR-1.4.2	Актуальность
BR-1.4.3	Приоритизация
BR-1.4.4	Ограничение видимости
BR-1.4.5	Кэширование
1.5 Ограничения и валидации
Тип	Ограничение
[REQUIRED]	Максимум 10 виджетов на рабочем столе
[REQUIRED]	Пагинация: не более 20 элементов на странице
[REQUIRED]	Поиск возвращает не более 100 результатов
[CONDITIONAL]	Если нет задач — показывать приветственное сообщение
1.6 Связи с другими модулями
Модуль	Тип связи
Заявки (Requests)	Read
Рейсы (Trips)	Read
Уведомления (Notifications)	Read/Write
Пользователи (Admin)	Read
Справочники	Read
1.7 MVP Scope
Включено:
- REQUIRED Базовые виджеты: задачи, активные рейсы, уведомления
- REQUIRED Глобальный поиск (по заявкам, рейсам, клиентам)
- REQUIRED Быстрые действия (создать заявку, создать рейс)
- REQUIRED Персонализация состава виджетов (drag-n-drop)
- REQUIRED Notification badge в header
Отложено:
- DEFERRED Кастомные виджеты (пользовательские SQL-запросы)
- DEFERRED Dashboard sharing (расшарить dashboard коллеге)
- DEFERRED Scheduled snapshots (снимки dashboard на email)
- DEFERRED Мобильная версия рабочего стола
- DEFERRED Тёмная тема интерфейса

2. Заказы (Orders)
2.1 Назначение модуля
Модуль управления коммерческими заказами, включающий создание, отслеживание оплаты, привязку к договорам и формирование финансовой документации. Обеспечивает связь между коммерческой и логистической составляющими бизнеса.

10.3 Список KPI
KPI	Формула расчёта
Выполнение в срок	(Рейсы в срок / Все рейсы) × 100%
Загрузка флота	(Занято машин / Всего машин) × 100%
Конверсия заявок	(Подтверждённые / Все новые) × 100%
Процент срывов	(Отменённые / Все) × 100%
Среднее время обработки	Σ(Время обработки) / Кол-во
Активные рейсы	COUNT(status = 'active')
Просроченные задачи	COUNT(deadline < now AND status != 'done')
Выручка (квартал)	Σ(Сумма закрытых заказов)
