## 1. Назначение экрана

### 1.1 Описание

**Карточка заявки** — центральный экран операционной работы в CRM-системе. Предназначен для просмотра, редактирования и управления жизненным циклом заявки на перевозку груза.

### 1.2 Цели экрана

| Цель | Описание |
|------|---------|
| **Информирование** | Предоставление полной информации о заявке в одном месте |
| **Управление** | Изменение статуса, флагов, назначение ответственных |
| **Навигация** | Быстрый переход к связанным объектам (рейсы, заказы, клиенты) |
| **Коммуникация** | Добавление комментариев, прикрепление файлов |
| **Анализ** | Просмотр истории изменений и контрольных точек |

### 1.3 Контекст использования

| Сценарий | Пользователь | Цель |
|----------|-------------|------|
| Просмотр заявки | Менеджер | Ознакомление с деталями |
| Изменение статуса | Диспетчер | Перевод в следующий статус |
| Назначение рейса | Диспетчер | Привязка рейса к заявке |
| Редактирование данных | Диспетчер/Менеджер | Корректировка параметров |
| Клонирование | Менеджер по продажам | Создание похожей заявки |
| Отмена | Диспетчер/Администратор | Отмена заявки |

### 3.2 Статусы заявки

| Статус | Код | Цвет | Иконка | Описание |
|--------|-----|------|--------|----------|
| Новая | `new` | Синий | 🟢 | Заявка создана, ожидает обработки |
| Расчёт | `calculating` | Голубой | 🔵 | Идёт расчёт стоимости |
| Предложена | `offered` | Жёлтый | 🟡 | Предложение отправлено клиенту |
| Подтверждена | `confirmed` | Оранжевый | 🟠 | Клиент подтвердил заявку |
| В работе | `in_progress` | Фиолетовый | 🟣 | Назначен рейс, идёт перевозка |
| Завершена | `completed` | Зелёный | ✅ | Заявка полностью исполнена |
| Отменена | `cancelled` | Серый | ⚫ | Заявка отменена |

### 3.3 Допустимые переходы статусов

| Текущий статус | Возможные переходы | Целевой статус | Условие перехода |
|---------------|-------------------|----------------|-----------------|
| new | → calculating | Расчёт | Начата работа с заявкой |
| new | → cancelled | Отменена | Отмена заявки |
| calculating | → offered | Предложена | Расчёт готов, отправлен клиенту |
| calculating | → cancelled | Отменена | Отмена заявки |
| offered | → confirmed | Подтверждена | Клиент подтвердил |
| offered | → calculating | Расчёт | Возврат на доработку |
| offered | → cancelled | Отменена | Отмена заявки |
| confirmed | → in_progress | В работе | Назначен рейс |
| confirmed | → cancelled | Отменена | Отмена заявки |
| in_progress | → completed | Завершена | Рейс завершён |
| in_progress | → cancelled | Отменена | Отмена (редко) |
| completed | — | — | Конечный статус |
| cancelled | — | — | Конечный статус |

### 3.4 Информация в блоке статуса

| Элемент | Источник данных | Формат | Пример |
|---------|----------------|--------|--------|
| Текущий статус | `REQUEST.status` | Badge + текст | `🟡 В работе` |
| Время текущего статуса | `REQUEST.updated_at` | datetime | `20.03.2026 15:30` |
| Кто изменил | `REQUEST_STATUS_HISTORY.changed_by` | User.name | `Петров И.` |
| Предыдущий статус | История | Текст | `Подтверждена` |
| Время перехода | История | datetime | `20.03.2026 14:30` |

### 4.2 Список флагов

| Флаг | Код | Цвет | Иконка | Описание | Обязательность |
|------|-----|------|--------|----------|----------------|
| Срочная | `urgent` | Красный | 🔴 | Приоритетная обработка | По выбору |
| Хрупкий | `fragile` | Синий | 🔵 | Аккуратное обращение | По выбору |
| Негабарит | `oversize` | Оранжевый | 🟠 | Превышение стандартных габаритов | По выбору |
| Температурный | `temp_controlled` | Голубой | ❄️ | Контроль температуры | Если требуется t ≠ ambient |
| Опасный (ADR) | `hazmat` | Красный | ☣️ | Опасный груз | Если ADR ≠ None |
| Экспресс | `express` | Жёлтый | ⚡ | Доставка в сжатые сроки | По выбору |

### 4.3 Хранение флагов

Флаги хранятся в поле `REQUEST.flags` как JSONB:

```json
{
  "urgent": true,
  "fragile": false,
  "oversize": true,
  "temp_controlled": true,
  "temp_min": 2,
  "temp_max": 8,
  "hazmat": false,
  "adr_class": null,
  "express": false
}
```

### 4.4 Условия активации флагов

| Флаг | Активация | Деактивация | Связанные поля |
|------|-----------|-------------|----------------|
| `temp_controlled` | `REQUEST.temperature_from/temperature_to` заполнено | Оба поля пустые | temperature_from, temperature_to |
| `hazmat` | `CARGO_ITEM.adr_class` ≠ null | adr_class = null | CARGO_ITEM.adr_class |
| `oversize` | Ручная установка или авто | Ручная отмена | length, width, height |
| `fragile` | Ручная установка | Ручная отмена | — |
| `urgent` | Ручная установка | Приоритет → normal | priority = urgent |
| `express` | Ручная установка | Приоритет → normal | priority = express |

## 11. Бизнес-правила смены статусов

### 11.1 Правила переходов

| Переход | Условие | Действие при переходе |
|---------|---------|----------------------|
| new → calculating | Начата работа | `created_at` = now |
| calculating → offered | Расчёт готов | `calculated_at` = now |
| offered → confirmed | Подтверждение | `confirmed_at` = now |
| confirmed → in_progress | Назначен рейс | `TRIP.status = 'scheduled'` |
| in_progress → completed | Рейс завершён | Все `TRIP.status = 'completed'` |
| * → cancelled | Отмена | Указание причины |

### 11.2 Автоматические действия при смене статуса

| Статус | Действия |
|--------|----------|
| new | — |
| calculating | Отправка уведомления ответственному |
| offered | Отправка email клиенту с предложением |
| confirmed | Отправка уведомления диспетчеру |
| in_progress | Обновление виджетов, уведомление клиента |
| completed | Архивация, формирование акта, уведомление |
| cancelled | Отправка уведомления, освобождение ресурсов |

### 11.3 Запрещённые переходы

```
┌─ ЗАПРЕЩЁННЫЕ ПЕРЕХОДЫ ─────────────────────────────────────────────────────────┐
│                                                                                  │
│  ❌ completed → *              (конечный статус)                                 │
│  ❌ cancelled → *              (конечный статус)                                 │
│  ❌ new → completed            (пропуск статусов)                                 │
│  ❌ new → in_progress          (пропуск статусов)                                │
│  ❌ in_progress → offered      (обратный переход запрещён)                      │
│  ❌ calculating → completed    (пропуск статусов)                               │
│                                                                                  │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 11.4 Валидация перехода статуса

| Проверка | Условие | Сообщение ошибки |
|----------|---------|------------------|
| Переход возможен | Есть допустимый переход | `Недопустимый переход статуса` |
| Указан комментарий | Для перехода в `cancelled` | `Укажите причину отмены` |
| Заполнен расчёт | Для перехода `offered` | `Сначала выполните расчёт` |
| Назначен рейс | Для перехода `in_progress` | `Назначьте рейс для заявки` |
| Рейс завершён | Для перехода `completed` | `Завершите рейс перед закрытием` |

## 12. Ошибки, предупреждения, валидации

### 12.1 Валидация при загрузке

| Ошибка | Состояние UI | Сообщение | Действие |
|--------|--------------|-----------|----------|
| Заявка не найдена | Error state | `Заявка не найдена. Возможно, она была удалена.` | "Назад к списку" |
| Нет прав на просмотр | Error state | `У вас нет прав для просмотра этой заявки.` | "Назад к списку" |
| Ошибка сервера | Error state | `Не удалось загрузить данные заявки.` | "Повторить" |

### 12.2 Валидация при редактировании

| Поле | Правило | Сообщение |
|------|---------|-----------|
| Клиент | Обязательно | `Выберите клиента` |
| Клиент | Существует | `Выбранный клиент не найден` |
| Тип перевозки | Обязательно | `Выберите тип перевозки` |
| Приоритет | Обязательно | `Выберите приоритет` |
| Вес | > 0 | `Вес должен быть больше 0` |
| Объём | > 0 | `Объём должен быть больше 0` |
| Точки маршрута | ≥ 2 | `Добавьте точки загрузки и выгрузки` |
| Температура от | < temperature_to | `Мин. температура должна быть меньше макс.` |

### 12.3 Валидация при смене статуса

| Ситуация | Сообщение | Реакция |
|----------|-----------|---------|
| Недопустимый переход | `Недопустимый переход статуса из «В работе» в «Новая»` | Блокировка кнопки |
| Отсутствует причина отмены | `Укажите причину отмены заявки` | Блокировка кнопки |
| Нет назначенного рейса | `Для перевода в статус «В работе» необходимо назначить рейс` | Предложение создать рейс |

### 12.4 Предупреждения (Warnings)

| Предупреждение | Триггер | Сообщение | Реакция |
|----------------|---------|-----------|---------|
| Пересечение с другим рейсом | Назначение водителя/транспорта | `Водитель уже назначен на рейс в это время. Назначить принудительно?` | Confirm/Cancel |
| Просрочен дедлайн | `deadline < now` | `Внимание! Заявка просрочена` | Жёлтая подсветка |
| Превышен лимит | Сумма > credit_limit | `Сумма превышает кредитный лимит клиента` | Warning badge |
| Нет доступного транспорта | Нет подходящего | `Нет доступного транспорта для заданных требований` | Warning badge |

## Приложения

### A. Состояния экрана

| Состояние | Триггер | UI |
|-----------|---------|-----|
| **Loading** | Загрузка данных | Skeleton overlay |
| **Loaded** | Данные получены | Полный UI |
| **Editing** | Клик "Редактировать" | Inline/modal редактирование |
| **Saving** | Отправка формы | Spinner на кнопке, disabled inputs |
| **Error** | Ошибка загрузки/сохранения | Error state + retry |
| **Success** | Успешное действие | Toast notification |

### B. Keyboard shortcuts

| Комбинация | Действие |
|------------|----------|
| `E` | Редактировать |
| `S` | Изменить статус |
| `N` | Создать рейс |
| `Esc` | Закрыть модальное окно |
| `Ctrl/Cmd + S` | Сохранить (в режиме редактирования) |

### C. Требования к accessibility

| Требование | Реализация |
|------------|------------|
| ARIA labels | Все интерактивные элементы |
| Keyboard navigation | Tab, Enter, Escape |
| Focus management | Focus trap в модальных окнах |
| Screen reader | Live regions для уведомлений |
| Color contrast | WCAG AA (4.5:1) |
| Status announced | Новый статус зачитывается |
